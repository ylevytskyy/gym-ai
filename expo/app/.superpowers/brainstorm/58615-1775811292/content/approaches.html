Three Approaches for Exercise Image System

Prompt template design, generation workflow, and app integration

A

Comic Strip Prompt (Recommended)

Prompt: Generate a single horizontal comic-strip-style image with numbered panels (one per instruction step). The prompt template includes exercise name, step descriptions, and a fixed style block.

Splitting: Script that detects panel boundaries (whitespace/dividers between panels) and crops into individual square images.

Storage: Images saved as assets/images/exercises/{id}/step-{n}.png

App UI: Horizontal swipeable carousel with dot indicators, synced to the instruction text below.

Pros

  • One prompt per exercise (75 total)
  • Steps are spatially consistent within one generation
  • Easiest to maintain style consistency

Cons

  • Panel splitting may need manual adjustment
  • Gemini may struggle with many panels (5-6 steps)
B

Grid Layout Prompt

Prompt: Generate a 2x2 or 2x3 grid image showing all steps. Each cell is one step, read left-to-right, top-to-bottom.

Splitting: Simple grid-based cropping (uniform cell sizes) — very reliable splitting.

Storage: Same as A: assets/images/exercises/{id}/step-{n}.png

App UI: Same carousel as A.

Pros

  • Grid splitting is trivial (equal rectangles)
  • Gemini handles grids well
  • Consistent cell sizes guaranteed

Cons

  • Square grid cells may waste space for some step counts (e.g., 5 steps in a 2x3 = 1 empty cell)
  • Slightly less natural than comic strip flow
C

Per-Step Prompting with Style Lock

Prompt: Generate each step as a separate image. Use a detailed style-lock prefix to maintain consistency across all prompts.

Splitting: Not needed — each image is already individual.

Storage: Same path structure.

App UI: Same carousel.

Pros

  • No splitting/cropping needed
  • Each image gets full resolution
  • More control per step

Cons

  • 300-400 individual prompts to paste manually
  • Style drift between steps very likely
  • 5x the manual work
